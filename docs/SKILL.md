# LeetCode 刷题助手

## 何时使用
- 需要分析 LeetCode 题目的解题思路
- 选择合适的数据结构和算法
- 编写和优化代码实现
- 分析时间和空间复杂度
- 生成测试用例验证解法

## 流程

### 1. 题目分析
- 理解输入输出类型和约束
- 明确边界条件和特殊 case
- 复述问题以确认理解

### 2. 思路推导
- 分析暴力解法及其复杂度
- 识别关键 pattern（如滑动窗口、双指针、动态规划、BFS/DFS、二分查找等）
- 给出最优思路和备选方案
- 画出示意或状态转移图

### 3. 代码实现
- 选择编程语言（C / C++ 等）
- 遵循语言惯用写法
- 包含核心逻辑和边界处理
- 添加必要注释

### 4. 复杂度分析
- 时间复杂度（最好/最坏/平均）
- 空间复杂度（不包括输入）
- 如果可能，给出主定理或摊还分析

### 5. 测试验证
- 基本功能测试
- 边界值测试（空输入、单元素、极大值等）
- 压力/性能测试
- 使用GoogleTest测试框架运行验证

### 6. 扩展思考（可选）
- 如果输入规模变化如何调整
- 如果是流式数据如何处理
- 是否有 follow-up 问题
- 是否可以推广到其他类似问题

## 示例 Prompt

```
/leetcode 两数之和
/leetcode 146. LRU Cache
/leetcode 接雨水 用C语言实现并生成测试
```
