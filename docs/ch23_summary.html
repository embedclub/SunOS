一、核心概念：双层结构

编译后的网络以两层文件形式到达引擎：

  • Dispatch Descriptor（调度描述符） — FlatBuffer 格式，命名操作并绑定缓冲区，位于逻辑层
  • Hardware Container（硬件容器） — Mach-O 形状的文件，魔数 0xbeefface，包含寄存器写入程序和权重库

调度描述符在上，硬件容器在下，下层是驱动 DMA 引擎和乘法阵列的任务描述符寄存器组。

3.3 标准程序形状

Op0_Cast (输入格式转换) → Op1_AneInference (整个融合图) → Op2_Cast (输出格式转换)

Cast 负责主机半精度布局与引擎内部交织布局之间的转换；单个 AneInference 即整个融合图。

3.4 Schema 恢复方法

通过反编译运行时中的 E5Serializer 方法族恢复 FlatBuffer schema：

  • SerializeFunction → Function 表
  • SerializeOperation → Operation 表
  • SerializeOperand → Operand 表
  • SerializeIOPort → IOPort 表
  • SerializeOpAttrs<CastOpT> + 11 个模板实例化 → OpAttrs union + OpType 枚举

3.5 关键性质

  • 深度不变性：1 个操作与 6 个操作融合后都归约为单个 AneInference，大小相同
  • 跟踪调度数而非操作数：bridge 操作将图切为 3 段 → 3 个 AneInference，描述符变大
  • 描述符大小不随计算形状变化：内维 64 与内维 256 产生相同大小的描述符

4.1 维度硬件编码

  • 空间维度 15 位，通道维度 17 位（最大 131071）
  • 输出通道组大小 3 位，卷积组 13 位

4.2 重定位槽（7 个）

编译器不写入缓冲区基址，留为命名的重定位槽，由加载器填入设备地址：

重定位寄存器槽位
0x1344输入 Tile 读取基址
0x134a第二操作数读取基址
0x1442输出 Tile 写入基址
0x1554Kernel Bias 流
0x1558Kernel Post-Scale 流
0x155cKernel Palette-Lookup 流
0x1560Kernel Activation-Lookup 流

每个重定位是自定义条，写入解析后的值（设备地址、片上内存数据集 ID 或缓冲区偏移）到精确的描述符位域。

4.3 稀疏序列化

序列化器稀疏地发出寄存器映像：只有当寄存器的值与架构默认值不同时才写入流。因此小收缩仅产生 6 到 8 条记录，每条固定 44 字节。

4.4 按代版本化

描述符布局按芯片代版本化：M1 构建 v10，其他代使用不同版本，维度块、重定位寄存器地址和字段宽度在不同版本间移动。M1 v10 完全省略了 compute-cache DMA 引擎。

提交调用是异步的 selector 2，2376 字节参数结构包含程序令牌、序列号、QoS 对及输入/输出/中间面标识符数组。40 字节异步回复包含序列结果和回显令牌。

九、关键设计要点

  • 魔数 0xbeefface 是小端 CE FA EF BE，与真实 Mach-O（0xfeedfacf）不同；修补这 4 字节后 otool 和 rabin2 可解析该文件
  • __FVMLIB 窗口：输入输出张量声明，fileoff 和 filesize 均为 0，仅在提交时由内核映射到地址转换孔
  • 窗口对齐到 0x4000（16 KiB，地址转换单元页粒度），描述符和权重对齐到 0x40（64 字节，引擎 tile 粒度）
  • ZinAneTd 链表：通过 +0x1c 的 next-offset 字段链接，0x000 → 0x300 → 0x500 → terminator
  • 权重分 tile：权重按 min(8, lanes) 分 tile，每个填充到 64 字节倍数，通过重定位引用
  • Stabs 风格形状描述符：符号表中 type 0x20 的条目，编码为 s<stride><axis> 四轴字节步幅列表
  • 容器自描述：符号表包含 24 条元素类型目录（含范围编码）、每轴步幅帧、操作类选择器等
  • 构建信息横幅以纯文本记录编译器调用，含工具链版本、目标、缓存策略、fp8 溢出策略等
